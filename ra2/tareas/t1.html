Práctica evaluable: Área de un rectángulo

Aplicaremos el ciclo de análisis → diseño → implementación → pruebas y evaluaremos distintos IDEs generando ejecutables en varios lenguajes.

Algoritmia Multilenguaje CLI / GUI

Objetivo de la práctica

Desarrollar un programa que solicite base y altura (valores reales positivos) de un rectángulo y calcule su área = base × altura, validando las entradas.

  • Entrada: base, altura > 0
  • Proceso: validación y cálculo
  • Salida: área del rectángulo

Enunciado

Desarrollar un programa que solicite al usuario las dimensiones de un rectángulo (base y altura) y calcule su área. Los datos deben ser reales positivos. El programa validará que la base y la altura sean mayores que cero; en caso contrario, volverá a pedir el dato.

Entradas

  • Base: número real positivo.
  • Altura: número real positivo.

Salidas

Área del rectángulo (real positivo).

Proceso

  1. Solicitar base y validar > 0.
  2. Solicitar altura y validar > 0.
  3. Calcular área = base × altura.
  4. Mostrar el resultado.

Especificación del proceso con validación

1) Pedir base. Si base ≤ 0, volver a pedirla.
2) Pedir altura. Si altura ≤ 0, volver a pedirla.
3) área ← base × altura
4) Mostrar área

Sugerencia didáctica: dibujar un diagrama de flujo o N/S con un bucle mientras por cada validación.

Implementaciones de referencia

Guárdalas como base de pruebas

Python

.py 
def pedir_positivo(texto):
    while True:
        try:
            x = float(input(texto))
            if x > 0:
                return x
            print("Debe ser un número positivo.")
        except ValueError:
            print("Entrada no válida.")

base = pedir_positivo("Base: ")
altura = pedir_positivo("Altura: ")
area = base * altura
print(f"Área = {area}")

Java (CLI)

.java 
import java.util.*;
public class Rectangulo {
  static double pedirPositivo(Scanner sc, String etiqueta){
    double v;
    do{ System.out.print(etiqueta); v = sc.nextDouble(); } while(v <= 0);
    return v;
  }
  public static void main(String[] args){
    Scanner sc = new Scanner(System.in);
    double base = pedirPositivo(sc, "Base: ");
    double altura = pedirPositivo(sc, "Altura: ");
    System.out.println("Área = " + (base*altura));
  }
}

JavaScript (Node.js)

.mjs 
import readline from 'node:readline/promises';
const rl = readline.createInterface({ input: process.stdin, output: process.stdout });
async function pedirPositivo(msg){
  while(true){
    const v = Number(await rl.question(msg));
    if(!Number.isNaN(v) && v > 0) return v;
    console.log('Introduce un número positivo');
  }
}
const base = await pedirPositivo('Base: ');
const altura = await pedirPositivo('Altura: ');
console.log('Área =', base*altura);
rl.close();

2. Evalúa entornos integrados de desarrollo

Criterios de evaluación

  1. Instala IDEs propietarios y libres.
  2. Añade y elimina módulos/extensiones.
  3. Personaliza y automatiza el entorno (plantillas, snippets, tareas).
  4. Configura el sistema de actualización.
  5. Genera ejecutables a partir de código de distintos lenguajes en un mismo IDE.
  6. Genera ejecutables del mismo código en varios IDEs.
  7. Identifica características comunes y específicas de cada IDE.

Tareas a realizar

  1. Instala y documenta brevemente (capturas + versión):
    • VS Code (libre), IntelliJ IDEA Community / Eclipse / NetBeans (libres), y uno propietario a elegir (p.ej., IntelliJ Ultimate, PyCharm Pro, WebStorm).
  2. Implementa el ejercicio en 4 lenguajes (elige entre Python, Java, C, C#, JavaScript/Node, PHP, JSP). Usa validación positiva y muestra el resultado. Guarda el código en carpetas: src/<lenguaje>/.
  3. Genera ejecutables/artefactos por IDE:
    • Java: JAR con main definido (Maven/Gradle opcional).
    • C/C++: binario usando el compilador del IDE.
    • Python/Node/PHP: ejecutable empaquetado (p.ej., PyInstaller, pkg para Node, Phar en PHP) o script con run task.
  4. Añade/Elimina módulos (plugins/extensiones) útiles para cada IDE e indica su función (formateo, depuración, Git, lanzadores, snippets).
  5. Personaliza y automatiza: crea una plantilla de archivo con cabecera, un snippet para entrada-validación y una tarea de ejecución/compilación.
  6. Configura actualizaciones: deja habilitada la búsqueda automática o programada y anota dónde se configura.
  7. Prueba cruzada: genera el ejecutable del mismo código en al menos 2 IDEs distintos y comprueba resultados.
  8. Informe comparativo (1–2 páginas): tabla con características comunes/específicas, valoración personal y capturas clave.
